Гайд

У цьому гайді буде описано, як працювати з даною програмою. Будуть описані правила роботи з нею та можливі варіанти проблем та їх вирішення. Рекомендуємо прочитати його перед використанням - інакше ми не несемо відповідальності за працездатність цього додатка.

Інструкція з роботи з програмою

Програма призначена для обробки даних з файлів Excel — синхронізації цін, наявності, знижок та інших параметрів між прайс-листами та маркетплейсами.

Важливо: програма працює за артикулами, повне співпадіння обов'язкове. Навіть різниця в одному символі (наприклад, `K`(Кирилиця) та `К`(Латиниця)) критична.

Можливості

  1. Синхронізація цін за артикулом.
  2. Перенесення наявності та встановлення статусу "Готово до відправки".
  3. Встановлення візуальної знижки (Не справжньої).
  4. Перенесення реальних знижок із зазначенням дат (якщо підтримується магазином).
  5. Робота з множинними файлами джерелами та цільовими.

Рекомендації перед початком

  • Протестуйте програму на копії файлу.
  • Дотримуйтесь інструкцій, зазначених нижче.
  • Використовуйте підказки при наведенні на елементи інтерфейсу.
  • Усі файли Excel мають бути закриті перед запуском обробки.

Початок роботи

Вибір файлів для виконання завдань:

  1. Завантаження цільових файлів (Target)

    • Кнопка: Файл для змін
    • Використовуються для внесення змін.

      • Приклади:

      • Файл магазину, в який потрібно перенести зміни;
      • при пошуку відсутності - Файл для пошуку.

  2. Завантаження вихідних файлів (Source)

    • Кнопка: Файл джерело
    • Використовуються як джерела даних.

      • Приклади:

      • Прайс-лист;
      • Файл магазину, з якого потрібно витягти значення;
      • при пошуку відсутності - файл, який є прикладом (зазвичай прайс-лист).

Види можливих завдань:

Тут будуть посилання на перехід до опису завдання. Також ви можете просто прокрутити вниз, щоб подивитися

  • Перенесення цін
  • Округлення цін (перенесених цін)
  • Збільшення цін (усіх цін)
  • Перенесення наявності
  • Перенесення кількості
  • Перенесення знижок
  • Перенесення дат завершення/початку знижок (при підтримці їх магазином)
  • Пошук відсутніх товарів

Загальні правила або особливості

Усі правила стосуються лише перенесення з прайсу!

Ви можете змінювати колонки залежно від розміру. Але потрібно це робити кожного разу, коли у вас є зміщення

Приклад:

  • Не правильно

    Артикул Ціна Наявність якась колонка якась колонка якась колонка
    товар1 200 + ... ... ...
    якась колонка Артикул Наявність якась колонка Ціна якась колонка
    ... товар2 + ... 300 ...
    товар3 700 + ... ... ...

    • На виході буде -> Товар1 з ціною 200 в наявності, Товар2 з ціною 300 в наявності, товар3 - ігнорується

  • Правильно

    Артикул Ціна Наявність якась колонка якась колонка якась колонка
    товар1 200 + ... ... ...
    якась колонка Артикул Наявність якась колонка Ціна якась колонка
    ... товар2 + ... 300 ...
    ... ... ... ... ... ...

    На виході буде -> Товар1 з ціною 200 в наявності, Товар2 з ціною 300 в наявності

Детально по кожному

Усі правила стосуються лише перенесення з прайсу!

Перенесення цін:

Обов'язкові колонки:

  • "Артикул" або "Артикул комплекта" (для ціни комплекта)
  • "Ціна" або "Ціна комплекта" (для артикула комплекта)

Правила:

обов'язково мають бути такі колонки як "Артикул" і "Ціна" в одиничному екземплярі. Якщо додати ще будь-який з них, перенесеться з останньої колонки значення.

    Приклад:

  • Не правильно:

    Артикул Ціна Артикул
    Товар1 200 Товар2

    • На виході буде -> Товар2 з ціною 200

  • Правильно:

    Артикул Ціна
    Товар1 200
    Товар2 300
    ... ...

    На виході буде -> Товар1 з ціною 200, Товар2 з ціною 300

Можна об'єднувати Артикули та Артикули комплектів. Але за наявності колонки комплекта - відповідно мають бути й наступні колонки для товару!

Приклад:

  • Не правильно:

    Артикул Ціна Артикул комплекта
    Товар1 200 Товар2

    • На виході буде -> Товар1 з ціною 200. А товар 2 ігнорується

  • Правильно:

    Артикул Ціна Артикул комплекта Ціна комплекта
    Товар1 200 Товар2 300
    ... ... ... ...

    На виході буде -> Товар1 з ціною 200 і Товар2 з ціною 300

Також їх можна розміщувати в будь-якому зручному для вас порядку, але обов'язкова умова для артикула комплекта має бути ціна комплекта, і "артикул" має мати "ціну"

Приклад як може бути:
  • Артикул Артикул комплекта Ціна Ціна комплекта
    Товар1 Товар2 200 300
    ... ... ... ...
  • Ціна Ціна комплекта Артикул Артикул комплекта
    200 300 Товар1 Товар2
    ... ... ... ...

Округлення цін:

Воно відбувається для товарів, які були перенесені з джерела. Тобто немає в джерелі - немає і в файлі де змінюють

Збільшення цін:

Проходить по кожному товару та робить "знижку" методом збільшення старої ціни. (за відсутності старої ціни ) просто збільшує ціну товару.

Пошук відсутніх товарів

Обов'язкові колонки:

  • Артикул
  • Артикул комплекта (якщо з прайсу і є в ньому)

Правила:

Проходить по всіх листах файлу та шукає всі значення в стовпцях "артикул" та "артикул комплекта", якщо воно є. Як і в попередніх, може бути лише в одиничному екземплярі на лист

Приклади:

  • Не правильно:

    Артикул Артикул
    Товар1 Товар2

    • На виході буде -> Товар1 ігнорується. товар2 буде включений у розрахунок пошуку

  • Правильно:

    Артикул
    Товар1
    Товар2
    ...

    • На виході буде -> Товар1 і товар2 буде включений у розрахунок пошуку

  • Артикул Артикул комплекта
    Товар1 Товар2
    ... ...

    • На виході буде -> Товар1 і товар2 буде включений у розрахунок пошуку
