Гайд

В этом гайде будет описано как работать с данной программой. Будут описаны правила работы с ней и возможные варианты проблем и их решения. Рекомендуем прочитать его перед использованием - иначе мы не в ответе за работоспособность этого приложения.

Инструкция по работе с программой

Программа предназначена для обработки данных из файлов Excel — синхронизации цен, наличия, скидок и других параметров между прайс-листами и маркетплейсами.

Важно: программа работает по артикулам, полное совпадение обязательно. Даже разница в одном символе (например, `K`(Кириллица) и `К`(Латиница)) критична.

Возможности

  1. Синхронизация цен по артикулу.
  2. Перенос наличия и установка статуса "Готово к отправке".
  3. Установка визуальной скидки (Не настоящей).
  4. Перенос реальных скидок с указанием дат (если поддерживается магазином).
  5. Работа с множественными файлами источниками и целевыми.

Рекомендации перед началом

  • Протестируйте программу на копии файла.
  • Следуйте инструкциям, указанным ниже.
  • Используйте подсказки при наведении на элементы интерфейса.
  • Все файлы Excel должны быть закрыты перед запуском обработки.

Начало работы

Выбор файлов для выполнения задач:

  1. Загрузка целевых файлов (Target)

    • Кнопка: Файл для изменений
    • Используются для внесения изменений.

      • Примеры:

      • Файл магазина в который нужно перенести изменения;
      • при поиске отсутствия - Файл для поиска.

  2. Загрузка исходных файлов (Source)

    • Кнопка: Файл источник
    • Используются как источники данных.

      • Примеры:

      • Прайс лист;
      • Файл магазина из которого нужно извлечь значения;
      • при поиске отсутствия - файл который является примером (обычно прайс лист).

Общие правила или особенности

Все правила относятся только к переносу из прайса!

Вы можете изменять колонки в зависимости от размера. Но нужно это делать каждый раз когда у вас идёт смещение

Пример:

  • Не правильно

    Артикул Цена Наличие какая-то колонка какая-то колонка какая-то колонка
    товар1 200 + ... ... ...
    какая-то колонка Артикул Наличие какая-то колонка Цена какая-то колонка
    ... товар2 + ... 300 ...
    товар3 700 + ... ... ...

    • На выходе будет -> Товар1 с ценой 200 в наличии, Товар2 с ценой 300 в наличии, товар3 - игнор

  • Правильно

    Артикул Цена Наличие какая-то колонка какая-то колонка какая-то колонка
    товар1 200 + ... ... ...
    какая-то колонка Артикул Наличие какая-то колонка Цена какая-то колонка
    ... товар2 + ... 300 ...
    ... ... ... ... ... ...

    На выходе будет -> Товар1 с ценой 200 в наличии, Товар2 с ценой 300 в наличии

Подробно по каждому

Все правила относятся только к переносу из прайса!

Перенос цен:

Обязательные колонки:

  • "Артикул" или "Артикул комплекта" (для цены комплекта)
  • "Цена" или "Цена комплекта" (для артикула комплекта)

Правила:

обязательно должны быть такие колонки как "Артикул" и "Цена" в единичном экземпляре. Если добавить ещё любой из них перенесётся из последней колонки значения.

    Пример:

  • Не правильно:

    Артикул Цена Артикул
    Товар1 200 Товар2

    • На выходе будет -> Товар2 с ценой 200

  • Правильно:

    Артикул Цена
    Товар1 200
    Товар2 300
    ... ...

    На выходе будет -> Товар1 с ценой 200, Товар2 с ценой 300

Можно обьеденять Артикулы и Артикулы комплектов. Но при наличии колонки комплекта - соответственно должна быть и последующие колонки для товара!

Пример:

  • Не правильно:

    Артикул Цена Артикул комплекта
    Товар1 200 Товар2

    • На выходе будет -> Товар1 с ценой 200. А товар 2 проигнорируется

  • Правильно:

    Артикул Цена Артикул комплекта Цена комплекта
    Товар1 200 Товар2 300
    ... ... ... ...

    На выходе будет -> Товар1 с ценой 200 и Товар2 с ценой 300

Так же их можно помещать в любом удобном для вас порядке, но обязательное условие для артикула комплекта должна быть цена комплекта и "артикул" должен иметь "цену"

Пример как может быть:
  • Артикул Артикул комплекта Цена Цена комплекта
    Товар1 Товар2 200 300
    ... ... ... ...
  • Цена Цена комплекта Артикул Артикул комплекта
    200 300 Товар1 Товар2
    ... ... ... ...

Округление цен:

Оно происходит для товаров которые были перенесены из источника. Тобишь нет в источнике - нет и в файле где изменяют

Увеличение цен:

Проходит по каждому товару и делает "скидку" методом увеличения старой цены. (при отсутствии старой цены ) просто увеличивает цену товару.

Поиск отсутствующих товарок

Обязательные колонки:

  • Артикул
  • Артикул комплекта (если из прайса и есть в нём)

Правила:

Проходится по всем листам файла и ищет все значения в столбцах "артикул" и "артикул комплекта" если оно есть. как и в предыдущих может быть только в единичном экземпляре на лист

Примеры:

  • Не правильно:

    Артикул Артикул
    Товар1 Товар2

    • На выходе будет -> Товар1 проигнорирован. товар2 будет включён в расчёт поиска

  • Правильно:

    Артикул
    Товар1
    Товар2
    ...

    • На выходе будет -> Товар1 и товар2 будет включён в расчёт поиска

  • Артикул Артикул комплекта
    Товар1 Товар2
    ... ...

    • На выходе будет -> Товар1 и товар2 будет включён в расчёт поиска
