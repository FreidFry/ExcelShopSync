Guide

This guide describes how to work with this program. It covers the rules of operation and possible issues with their solutions. We recommend reading it before use - otherwise, we are not responsible for the application's functionality.

Program Usage Instructions

The program is designed to process data from Excel files — synchronizing prices, availability, discounts, and other parameters between price lists and marketplaces.

Important: the program works by article numbers, exact match is mandatory. Even a difference in one character (e.g., `K` (Cyrillic) and `К` (Latin)) is critical.

Features

  1. Price synchronization by article number.
  2. Transfer of availability and setting the "Ready to ship" status.
  3. Setting visual discounts (not real).
  4. Transfer of real discounts with date specifications (if supported by the store).
  5. Working with multiple source and target files.

Recommendations Before Starting

  • Test the program on a file copy.
  • Follow the instructions provided below.
  • Use tooltips when hovering over interface elements.
  • All Excel files must be closed before starting processing.

General Rules or Features

All rules apply only to transfers from the price list!

You can change columns depending on the size. But this needs to be done every time there is an offset.

Example:

  • Incorrect

    Article number Price Availability Some column Some column Some column
    product1 200 + ... ... ...
    Some column Article number Availability Some column Price Some column
    ... product2 + ... 300 ...
    product3 700 + ... ... ...

    • Output will be -> Product1 with price 200 in stock, Product2 with price 300 in stock, product3 - ignored

  • Correct

    Article number Price Availability Some column Some column Some column
    product1 200 + ... ... ...
    Some column Article number Availability Some column Price Some column
    ... product2 + ... 300 ...
    product3 700 + ... ... ...

    Output will be -> Product1 with price 200 in stock, Product2 with price 300 in stock

Details for Each

All rules apply only to transfers from the price list!

Price Transfer:

Mandatory columns:

  • "Article number" or "Kit article number" (for kit price)
  • "Price" or "Kit price" (for kit article number)

Rules:

There must be columns like "Article number" and "Price" in a single instance. If you add more of them, the values will be taken from the last column.

    Example:

  • Incorrect:

    Article number Price Article number
    Product1 200 Product2

    • Output will be -> Product2 with price 200

  • Correct:

    Article number Price
    Product1 200
    Product2 300
    ... ...

    Output will be -> Product1 with price 200, Product2 with price 300

You can combine Article numbers and Kit article numbers. But if there is a kit column, there must be corresponding columns for the product!

Example:

  • Incorrect:

    Article number Price Kit article number
    Product1 200 Product2

    • Output will be -> Product1 with price 200. Product 2 will be ignored

  • Correct:

    Article number Price Kit article number Kit price
    Product1 200 Product2 300
    ... ... ... ...

    Output will be -> Product1 with price 200 and Product2 with price 300

They can also be placed in any order convenient for you, but the mandatory condition is that for a kit article number there must be a kit price and "article number" must have "price"

Example of how it can be:
  • Article number Kit article number Price Kit price
    Product1 Product2 200 300
    ... ... ... ...
  • Price Kit price Article number Kit article number
    200 300 Product1 Product2
    ... ... ... ...

Price Rounding:

It occurs for products that were transferred from the source. Meaning, if it's not in the source, it won't be in the file being modified.

Price Increase:

Goes through each product and applies a "discount" by increasing the old price. (If there is no old price) it simply increases the product's price.

Search for Missing Products

Mandatory columns:

  • Article number
  • Kit article number (if from price list and present in it)

Rules:

Goes through all sheets of the file and searches for all values in "article number" and "kit article number" columns if they exist. As in previous cases, there can only be one instance per sheet.

Examples:

  • Incorrect:

    Article number Article number
    Product1 Product2

    • Output will be -> Product1 ignored. product2 will be included in the search calculation

  • Correct:

    Article number
    Product1
    Product2
    ...

    • Output will be -> Product1 and product2 will be included in the search calculation

  • Article number Kit article number
    Product1 Product2
    ... ...

    • Output will be -> Product1 and product2 will be included in the search calculation
